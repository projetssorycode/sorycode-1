Server

Nutze den sorycode-Server ueber HTTP.

Der Befehl sorycode serve startet einen headless HTTP-Server. Er stellt einen OpenAPI-Endpunkt bereit, den ein sorycode-Client nutzen kann.

Nutzung

Optionen

FlagDescriptionDefault
--portPort to listen on4096
--hostnameHostname to listen on127.0.0.1
--mdnsEnable mDNS discoveryfalse
--mdns-domainCustom domain name for mDNS servicesorycode.local
--corsAdditional browser origins to allow[]

Authentifizierung

Setze SORYCODE_SERVER_PASSWORD, um den Server mit HTTP Basic Auth zu schuetzen. Der Benutzername ist standardmaessig sorycode, kann aber mit SORYCODE_SERVER_USERNAME ueberschrieben werden. Das gilt fuer sorycode serve und sorycode web.

Funktionsweise

Wenn du sorycode startest, werden TUI und Server gestartet. Die TUI ist dabei der Client, der mit dem Server spricht; dessen OpenAPI-3.1-Endpunkt dient auch zur Generierung des SDK.

Tipp

Nutze den sorycode-Server, um sorycode programmatisch anzusteuern.

Diese Architektur erlaubt mehrere Clients und programmatische Nutzung.

Mit sorycode serve startest du einen eigenstaendigen Server. Laeuft bereits eine TUI, startet sorycode serve trotzdem eine neue Serverinstanz.

Mit bestehendem Server verbinden

Beim TUI-Start werden Port und Hostname zufaellig gewaehlt. Alternativ gibst du --hostname und --port als Flags vor und verbindest dich dann gezielt damit.

Der Endpunkt /tui kann die TUI ueber den Server steuern, etwa zum Vorbelegen oder Abschicken von Prompts. Dieses Muster nutzen auch die SoryCode-IDE-Plugins.
