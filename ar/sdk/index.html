مجموعة تطوير البرمجيات (SDK)

عميل JavaScript آمن الأنواع لخادم sorycode.

توفر SDK الخاصة بـ sorycode لـ JS/TS عميلا آمنا للأنواع للتفاعل مع الخادم. استخدمها لبناء التكاملات والتحكم في sorycode برمجيا.

اعرف المزيد حول كيفية عمل الخادم. للاطلاع على أمثلة، تفقد المشاريع التي أنشأها المجتمع.

الأخطاء

يمكن أن ترمي SDK أخطاء يمكنك التقاطها ومعالجتها:

المخرجات المنظمة

يمكنك طلب مخرجات JSON منظمة من النموذج عن طريق تحديد format مع مخطط JSON. سيستخدم النموذج أداة StructuredOutput لإرجاع JSON تم التحقق من صحته ومطابق للمخطط الخاص بك.

الاستخدام الأساسي

أنواع صيغ الإخراج

النوعالوصف
textالافتراضي. استجابة نصية قياسية (بدون مخرجات منظمة)
json_schemaيرجع JSON تم التحقق من صحته ومطابق للمخطط المقدم

صيغة مخطط JSON

عند استخدام type: 'json_schema'، يجب تقديم:

الحقلالنوعالوصف
type'json_schema'مطلوب. يحدد وضع مخطط JSON
schemaobjectمطلوب. كائن مخطط JSON الذي يحدد بنية الإخراج
retryCountnumberاختياري. عدد محاولات إعادة التحقق (الافتراضي: 2)

معالجة الأخطاء

إذا فشل النموذج في إنتاج مخرجات منظمة صالحة بعد جميع المحاولات، ستتضمن الاستجابة StructuredOutputError:

أفضل الممارسات

  1. قدّم أوصافا واضحة في خصائص المخطط لمساعدة النموذج على فهم البيانات التي يجب استخراجها
  2. استخدم required لتحديد الحقول التي يجب أن تكون موجودة
  3. حافظ على تركيز المخططات - المخططات المتداخلة المعقدة قد تكون أصعب على النموذج لملئها بشكل صحيح
  4. عيّن retryCount مناسبا - قم بزيادته للمخططات المعقدة، وتقليله للمخططات البسيطة

الملفات

الطريقةالوصفالاستجابة
find.text({ query })البحث عن نص داخل الملفاتمصفوفة من كائنات المطابقة مع path وlines وline_number وabsolute_offset وsubmatches
find.files({ query })العثور على الملفات والمجلدات بالاسمstring[] (مسارات)
find.symbols({ query })العثور على رموز مساحة العملSymbol[]
file.read({ query })قراءة ملف{ type: "raw" | "patch", content: string }
file.status({ query? })جلب حالة الملفات المتتبَّعةFile[]

يدعم find.files بعض حقول الاستعلام الاختيارية:

  • type: "file" أو "directory"
  • directory: تجاوز جذر المشروع لعملية البحث
  • limit: الحد الأقصى للنتائج (1–200)
